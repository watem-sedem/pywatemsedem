pywatemsedem modelinput and modeloutput

1. Introduction

This tutorial describes the use of Modelinput and Modeloutput class of the pywatemsedem Python package. Via these classes the input and output of a WaTEM-SEDEM model run can be dynamically loaded into memory for e.g. analysis or plotting. Pywatemsedem also contains a postprocessing module, but this module is explained in another tutorial. The postprocessing module makes use of the modelinput and modeloutput classes, described in this tutorial.

3. Modelinput and Modeloutput objects

3.1. Initialisation

The object initialisation arguments can be fetched based on ini file and the epsg code

[ ]:

ini = results_folder / "modelinput/inifile.ini"
epsg = 31370

from pywatemsedem.geo.utils import get_rstparams
_, rp = get_rstparams(ini, epsg=epsg)
resolution = rp["res"]
nodata = rp["nodata"]

The Modelinput and Modeloutput objects can be initialised with the following commands

[ ]:

from pywatemsedem.io.modelinput import Modelinput
modelinput = Modelinput(ini, resolution, epsg, nodata)

from pywatemsedem.io.modeloutput import Modeloutput
modeloutput = Modeloutput(ini, resolution, epsg, nodata)

3.2. Map variables

The modelinput and modeloutput objects contain several properties that are RasterMemory objects (see documentation of RasterMemory for more information). These properties can be accessed by their name, which is the same as the variable name in the modelinput and modeloutput. The variable names are listed below.

  • Properties of the modelinput object:

    “mask”,”rivermask”, “buffers”, “cfactor”, “dtm”, “kfactor”, “ktc”, “outlet”, “pfactor”, “compositelanduse”, “ptef”, “riversegments”, “riverrouting”

  • Properties of the modeloutput object:

    “aspect”, “capacity”, “cumulative”, “ls”, “rusle”, “sedi_export”, “sedi_in”, “sedi_out”, “slope”, “uparea”, “watereros_kg”, “watereros_mm”

As you can see in the lists above, the modelinput object also contains a mask and a rivermask, derived from the model input files.

We will use the rivermask as an example

[ ]:

rivermask = modelinput.rivermask

To see what datatype rivermask is, we symply print it

[ ]:

rivermask

As you can see, it is a RasterFile-object. A RasterFile-object has some properties, such as the rasterproperies, a plot function, and an arr property that contains the raster values as a numpy array. We can check these properties for the rivermask.

[ ]:

rivermask.arr

[ ]:

rivermask.rp.rasterio_profile

[ ]:

rivermask.plot()

[ ]:

rivermask.histogram(nodata=nodata)

We can also write the rivermask to a new file, the write function of the RasterFile-object can be used for this.

The above steps can also be done for the other RasterMemory properties of the modelinput and modeloutput objects, such as the buffers property of the modelinput object.

[ ]:

modelinput.buffers.plot()

Or for the sedi_out property of the modeloutput object.

[ ]:

modeloutput.sedi_out.plot()

3.3 Table variables

The modelinput and modeloutput objects can also contain tables. Following tables are available as pandas dataframes:

  • Modelinput:

    “adjacent_segments”, “upstream_segments”

  • Modeloutput:

    “cumulative_sediment_segments”, “routing_missing”, “routing”, “total_sediment_segments”, “total_sediment”,

[ ]:

modeloutput.routing

As mentioned above, these objects are only used for io-operations. For analysis and derivivng information from these objects, the postprocessing module should be used.
